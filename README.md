Для 3 задания была выбрана предметная область:
Система автоматической оценки кредитных заявок в банковской сфере

Данная предметная область охватывает процесс рассмотрения заявок на получение кредита в финансовом учреждении. Система анализирует характеристики клиентов и параметры запрашиваемых кредитов для автоматического принятия решений о кредитовании.

Онтология включает следующие основные компоненты:

Классы:
Клиент - физическое лицо, подающее кредитную заявку
Кредитная_заявка - запрос на получение кредита
Кредитный_продукт - виды кредитов (потребительский, автокредит, ипотека и др.)
Кредитная_история - данные о кредитной дисциплине клиента
Работа - информация о трудоустройстве клиента
Уровень_риска - категории риска кредитных продуктов
Категории для стажа, дохода и суммы кредита

Свойства (предикаты):
подал_заявку - связь между клиентом и заявкой
имеет_историю - связь клиента с кредитной историей
имеет_работу - связь клиента с информацией о работе
запрашивает_продукт - связь заявки с типом кредита
имеет_уровень_риска - связь кредитного продукта с уровнем риска

Категориальные свойства для стажа, дохода и суммы

Задача классификации
Автоматическое определение статуса кредитных заявок - отнесение каждой заявки к одному из двух классов:
Одобренная_заявка - заявка удовлетворяет критериям одобрения
Отклоненная_заявка - заявка не удовлетворяет критериям одобрения

Вопрос для классификации
"Можно ли отнести данную кредитную заявку к классу 'Одобренная_заявка' или 'Отклоненная_заявка'?"

Классификация выполняется на основе анализа:
Кредитной истории клиента (плохая/хорошая/отличная)
Стажа работы и уровня дохода
Типа запрашиваемого кредитного продукта и его уровня риска
Запрашиваемой суммы кредита

Для реализации классификации разработаны SWRL-правила, которые формализуют бизнес-логику банка, например:
Заявки клиентов с хорошей кредитной историей и хорошим стажем одобряются
Заявки клиентов с плохой кредитной историей отклоняются
Клиенты с малым стажем не могут получать крупные кредиты
